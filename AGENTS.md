## Skill Loading

Before substantial work:
- Skill check: run `npx @tanstack/intent@latest list`, or use skills already listed in context.
- Skill guidance: if one local skill clearly matches the task, run `npx @tanstack/intent@latest load <package>#<skill>` and follow the returned `SKILL.md`.
- Monorepos: when working across packages, run the skill check from the workspace root and prefer the local skill for the package being changed.
- Multiple matches: prefer the most specific local skill for the package or concern you are changing; load additional skills only when the task spans multiple packages or concerns.

## Architecture notes

- File-based routes live in `src/routes`.
- `src/routes/__root.tsx` owns the document shell and must render `HeadContent` in `<head>` and `Scripts` in `<body>`.
- `src/router.tsx` creates the router from `src/routeTree.gen.ts`.
- `src/routeTree.gen.ts` is generated by TanStack Router CLI. Do not hand-edit it; run `npm run generate-routes`.
- In `vite.config.ts`, keep `tanstackStart()` before `viteReact()` as required by TanStack Start guidance.
- TanStack Start code is isomorphic by default. Use `createServerFn` or server-only helpers before adding server-only logic such as secrets, database access, or filesystem access.
- `verbatimModuleSyntax` is disabled because the installed Start skill warns it can leak server bundles into client bundles.

## Environment variables

- No environment variables are required for the current blank app.
- Client-exposed variables must use Vite's `VITE_` prefix.
- Keep secrets server-only; do not expose them through `VITE_` variables.

## Development and deployment

- Install dependencies: `npm install`.
- Start local dev server: `npm run dev`.
- Generate route tree after route changes: `npm run generate-routes`.
- Build for production: `npm run build`.
- Preview production build: `npm run preview`.
- The default production start target can be added when a deployment target is chosen. For Node deployment, TanStack Start commonly runs the built server output with Node.

## Known gotchas

- The TanStack CLI and Intent commands require npm registry access.
- The project root is the actual app. `my-tanstack-app` was the original CLI scaffold directory used as the source for merging.
- Do not reintroduce Tailwind, Devtools, sample routes, or feature scaffolding unless explicitly requested.
